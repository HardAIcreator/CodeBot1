# CodeBot 🤖

Telegram-бот, который пишет код, создаёт проекты и отправляет готовые файлы прямо в чат — аналог Cursor / Bolt.new в Telegram.

## Возможности

- ✅ Пишет код по текстовому описанию
- ✅ Создаёт проекты из нескольких файлов и упаковывает их в ZIP
- ✅ Объясняет код и исправляет ошибки
- ✅ Помнит историю диалога (SQLite)
- ✅ Базовая проверка Python-кода на синтаксические ошибки перед отправкой

## Стек

- Python 3.13
- aiogram 3
- OpenAI API (модель задаётся в `.env`)
- SQLite (через `aiosqlite`)
- Docker (опционально)

## Структура проекта

```
CodeBot/
├── main.py                 # точка входа, DI зависимостей, запуск polling
├── config.py                # загрузка конфигурации из переменных окружения
├── database.py               # слой SQLite: пользователи + история диалога
├── handlers/
│   ├── commands.py            # /start, /help, /clear, /code
│   └── generate.py            # основная логика генерации и отправки файлов
├── services/
│   ├── ai.py                   # обёртка над OpenAI API, JSON-протокол ответа
│   └── file_builder.py          # сохранение файлов на диск, упаковка в ZIP
├── utils/
│   └── code_check.py             # проверка синтаксиса Python через ast
├── generated/                     # сюда сохраняются файлы пользователей (создаётся автоматически)
├── requirements.txt
├── Dockerfile
└── .env.example
```

## Запуск

1. Установить зависимости:

   ```bash
   pip install -r requirements.txt
   ```

2. Скопировать `.env.example` в `.env` и заполнить токены:

   ```bash
   cp .env.example .env
   ```

   - `BOT_TOKEN` — получить у [@BotFather](https://t.me/BotFather)
   - `OPENAI_API_KEY` — получить на [platform.openai.com](https://platform.openai.com)

3. Запустить переменные окружения и бота:

   ```bash
   export $(cat .env | xargs)   # на Windows используйте python-dotenv или set
   python main.py
   ```

### Через Docker

```bash
docker build -t codebot .
docker run --env-file .env codebot
```

## Как это работает

1. Пользователь пишет любое сообщение или команду `/code <описание>`.
2. Бот отправляет историю диалога + запрос в OpenAI с системным промптом,
   который заставляет модель отвечать строго в JSON: `{"summary": ..., "files": [...]}`.
3. `FileBuilder` сохраняет файлы на диск в `generated/<user_id>/`.
4. Если файл один — отправляется как документ. Если несколько — упаковывается в `project.zip`.
5. Python-файлы дополнительно проверяются на синтаксические ошибки (`ast.parse`),
   и если есть проблема — бот покажет номер строки и текст ошибки, не выполняя код.

## Важно про безопасность

Бот **не выполняет** сгенерированный код — только проверяет синтаксис Python статически
(`ast.parse`, без `exec`/`eval`). Реальный запуск кода в продакшене нужно делать только
в изолированной песочнице (например, отдельный Docker-контейнер без доступа к сети и диску хоста).
